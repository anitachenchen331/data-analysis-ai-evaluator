Codex Skill Architecture

Data Analysis AI Evaluator

把数据分析类 AI 产品评测拆成可复用流程：先确认能力与边界，再隔离运行实验，最后用内部答案材料评分。

适用于数据问答、SQL/查询生成、语义层、图表/看板、异动诊断、归因分析、自动报告与权限治理评测。

评测工作流

Human-in-the-loop gates protect fairness and scope
1

产品调研

读取公开资料或用户材料，整理产品能力、来源可信度和待确认项。

2

目标确认

确认评测目的、能力范围、数据场景、权限和版本，避免测错方向。

3

实验组设计

定义对照组、产品组、上下文组、工具限制和禁止读取材料。

4

蓝图与题集

生成数据场景、指标口径、陷阱、问题矩阵和公开任务。

5

隔离运行

各组只读取允许材料，记录输出、工具证据、耗时、错误与修复过程。

6

评分对比

评分阶段才读取内部 key、标准 SQL、答案 JSON 和结果文件。

7

报告沉淀

输出结论、失败类型和下一轮建议；经验写回前必须再次确认。

Skill 包里有什么

  • SKILL.md: 触发条件和总流程
  • references/: 每一步怎么做
  • assets/: 可复制的评测模板

哪些地方必须人确认

  • 测什么产品、什么版本
  • 实验组是否公平隔离
  • 题目、口径和评分权重
  • 标准答案有争议就暂停
运行一次评测会生成的文件边界

给实验组看

公开题集 每组 prompt 结果填写模板

评分时才看

内部评分 key 标准 SQL / 校验逻辑 标准答案 JSON

运行后记录

每组最终回答 工具调用、耗时、错误 实际读取了哪些文件

最后交付

总分和逐题评分 失败类型和关键案例 可复用经验候选

为什么要分开这些文件

  • 防止模型偷看答案
  • 防止不同实验组互相污染
  • 让评分依据可追溯
  • 让结论能解释、能复跑

脚本负责哪些脏活

  • 检查评测包结构
  • 扫描公开文件泄漏
  • 用标准 SQL 生成答案
  • 生成评分工作表草稿
