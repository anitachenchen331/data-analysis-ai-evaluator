# Human-in-the-loop Gates

## 判定原则

如果某一步不让用户确认，后续工作可能整体方向错误、实验不公平、评分不可解释或结论不可用，就必须停下来确认。

## 必停节点

| Gate | 必须确认内容 | 不确认的风险 |
|---|---|---|
| G1 产品画像 | 产品名称、版本、公开能力、可信度、本轮评测目标 | 测错产品或测错能力 |
| G2 实验设计 | 实验组、对照组、允许和禁止的信息与工具 | 实验污染或对照不公平 |
| G3 评测蓝图 | 数据场景、核心对象、指标口径、问题矩阵 | 数据、题目和答案整体偏离目标 |
| G4 评分体系 | 评分维度、权重、自动评分和人工评分边界 | 最终胜负无法解释 |
| G5 工具降级 | 产品入口、连接、执行、渲染、记忆等关键能力不可用 | 实验含义变化 |
| G6 标准争议 | 标准答案、口径、题目或评分规则有争议 | 错误结论被固化 |
| G7 经验沉淀 | 候选经验是否写回 skill 或模板 | 单次经验污染通用 skill |

## 非必停但建议确认

- 是否生成对外分享版报告。
- 是否重跑稳定性实验。
- 是否调整题目数量或难度。
- 是否新增数据场景。

## 标准引导句

```text
这里必须先确认，因为如果这个判断错了，后续题集、标准答案和评分都会跟着错。
```

```text
我可以给出默认推荐，但需要你确认是否接受这个实验边界。
```
