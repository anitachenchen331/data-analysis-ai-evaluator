# 通用评测工作流

## 目标

把数据分析类 AI 产品评测拆成可复用的八个阶段：产品调研、实验设计、评测蓝图、资产生成、实验运行、评分对比、结论输出、经验沉淀。

## 阶段 1：产品调研与目标确认

输入：

- 产品名称、链接、文档、代码仓库、截图或自然语言描述。
- 用户初步目标。

处理：

- 主动调研产品公开能力。
- 整理能力主张、证据来源和可信度。
- 区分可测能力、暂不可测能力和需要用户补充的信息。

输出：

- `product_eval_brief.md`

必须停下来确认：

- 产品识别是否正确。
- 本轮评测哪些能力。
- 实验目的是什么。

下一步引导：

```text
产品画像确认后，我会按你的实验目的设计最小可行且有解释力的实验组。
```

## 阶段 2：实验组设计

输入：

- 已确认产品画像。
- 实验目的。
- 可用产品入口和权限。

处理：

- 根据目标设计实验组，不固定套模板。
- 为每组定义允许信息、禁止信息、允许工具、禁止工具、输出路径和证据要求。

输出：

- `experiment_and_scoring_plan.md`
- `group_prompts.md`

必须停下来确认：

- 实验组是否公平。
- 文件边界是否清楚。
- 是否需要记录耗时、成本和轮次。

下一步引导：

```text
实验组确认后，我会把数据场景、指标口径和问题矩阵合并成评测蓝图给你确认。
```

## 阶段 3：评测蓝图设计

输入：

- 产品能力。
- 数据场景。
- 输出形态。
- 用户关注的业务问题。

处理：

- 设计数据场景、核心对象、指标口径、敏感字段、陷阱类型和问题矩阵。
- 同时覆盖查询、图表、诊断、报告、治理等需要评测的产物类型。

输出：

- `evaluation_blueprint.md`

必须停下来确认：

- 数据场景是否合理。
- 核心指标和陷阱是否合理。
- 问题矩阵是否覆盖目标能力。

下一步引导：

```text
蓝图确认后，我会生成正式评测资产，包括公开题集、内部评分 key、标准答案和运行 prompt。
```

## 阶段 4：评测资产生成

输入：

- 已确认蓝图。
- 数据源或合成数据需求。
- 评分目标。

处理：

- 生成数据、数据字典、指标口径、公开题集、内部评分 key、标准答案、实验 prompt 和结果模板。
- 对公开题集做泄漏检查。

输出：

- 评测资产目录。

条件停点：

- 数据源不可用。
- 产品不支持目标数据源。
- 标准答案无法自动生成。

下一步引导：

```text
资产生成后，可以运行实验。你可以手动开独立窗口运行，也可以在环境允许时让我代为启动隔离任务。
```

## 阶段 5：实验运行

输入：

- 实验组 prompt。
- 公开题集。
- 目标产品或对照环境。
- 结果模板。

处理：

- 隔离运行各组。
- 记录实际读取文件、工具调用、耗时、错误、修复过程和最终产物。

输出：

- `results/<组名>_result.md`

条件停点：

- 关键工具、账号、权限或数据连接不可用。

下一步引导：

```text
所有实验组完成后，我会读取内部评分材料并生成逐题评分报告。
```

## 阶段 6：评分与对比

输入：

- 各组结果。
- 内部评分 key。
- 标准答案或人工校验规则。
- 评分权重。

处理：

- 逐题评分。
- 汇总能力模块得分。
- 分析失败类型。
- 区分模型能力、上下文能力和产品链路能力。

输出：

- `results/final_scoring_report.md`

条件停点：

- 标准答案或评分规则有争议。

下一步引导：

```text
评分完成后，我会提炼本轮可复用经验，但只有你确认后才会沉淀回 skill。
```

## 阶段 7：结论输出

输入：

- 总分、逐题分、链路分、失败类型、耗时。

处理：

- 输出胜负判断、能力画像、关键失败案例、实验限制和下一轮建议。

输出：

- 面向用户的最终评测报告。

## 阶段 8：经验沉淀

输入：

- 最终报告。
- 用户反馈。
- 失败类型和修正建议。

处理：

- 提取通用经验、项目经验和偶发问题。
- 先给用户看候选清单。
- 用户确认后再写回 skill 或项目资产。

输出：

- `learning_candidates.md`
- 经确认后的 skill 更新。
