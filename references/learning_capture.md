# 经验沉淀机制

## 原则

不要自动把单轮评测经验写回 skill。必须先提取候选经验，让用户确认哪些是通用经验，哪些只是当前项目经验。

## 分类

| 类型 | 处理方式 |
|---|---|
| 通用经验 | 用户确认后可写回 skill |
| 项目经验 | 写入当前项目评测说明，不写回通用 skill |
| 偶发问题 | 记录在本轮报告，不沉淀 |

## 候选经验清单

每条候选经验记录：

- 经验标题。
- 来源证据。
- 适用范围。
- 不适用场景。
- 建议写入位置。
- 是否需要改模板或脚本。

## Human-in-loop

沉淀前必须让用户确认：

- 哪些经验可以写回。
- 写回到哪个 reference、template 或 script。
- 是否需要保留项目特定表述。

## 引导语

```text
我整理了一份可沉淀经验候选清单。请确认哪些是通用经验，确认后我再写回 skill，避免把本轮项目特有现象污染成通用规则。
```
