# 问题矩阵设计

## 原则

一级分类按产品能力组织，不按 SQL 技巧组织。多表关联、分组、排序、topN、时间窗口属于基础查询能力的子类型。

## 通用能力题型

| 能力模块 | 典型题型 |
|---|---|
| 基础查询 | 单表查询、多表关联、过滤、分组、排序、topN、时间窗口 |
| 指标口径 | 多口径指标、口径版本选择、分子分母、排除规则、指标对比 |
| 语义理解 | 字段同义词、业务别名、实体识别、指标映射 |
| 图表生成 | 图表类型选择、字段映射、聚合粒度、标题与注释、误导性可视化识别 |
| 看板生成 | 指标卡、趋势图、分布图、明细表、筛选器、布局合理性 |
| 异动诊断 | 环比或同比变化、贡献拆解、异常定位、候选因素验证 |
| 归因分析 | 归因窗口、控制变量、分组对比、因果措辞边界 |
| 多轮分析 | 上下文继承、追问改写、口径保持、修正前一轮错误 |
| 自动报告 | 结论先行、证据链、图表引用、风险说明、可复核性 |
| 数据质量 | 缺失、重复、异常、延迟、口径冲突、样本不足 |
| 权限治理 | 敏感字段拒绝、脱敏聚合、角色权限、提示注入防护 |
| 产品链路 | 数据连接、schema 探索、语义检索、执行校验、错误修复、证据记录 |

## 题目字段

公开题集每题只包含：

- `task_id`
- `category`
- `module_under_test`
- `question`
- `response_mode`
- `expected_artifact_type`

公开题集不得包含：

- 标准答案。
- 标准查询。
- 标准逻辑。
- 常见错误。
- 评分提示。

## 题量建议

- 轻量评测：8-12 题。
- 标准评测：20-30 题。
- 完整产品链路评测：业务题 20-30 题，加链路任务 5-12 个。

## 质量检查

每轮至少包含：

- 一个模糊问题。
- 一个权限或治理问题。
- 一个可追溯性问题。
- 一个输出粒度问题。
- 一个口径冲突或版本问题。

如果评测图表、看板、诊断或报告，必须加入对应产物质量题。
