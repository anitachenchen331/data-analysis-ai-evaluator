# 实验结果与最终报告规范

## 实验结果字段

每组每题建议记录：

| 字段 | 说明 |
|---|---|
| 任务编号 | 题目唯一编号 |
| 问题 | 原始输入问题 |
| 题目类别 | 能力模块 |
| 运行阶段是否提供标准逻辑 | 固定写“未提供” |
| 运行阶段是否提供标准答案 | 固定写“未提供” |
| 模型输出 | 原始回答 |
| 生成查询 | SQL 或其他查询 |
| 生成产物 | 图表、报告、看板等 |
| 是否可执行或可渲染 | 是、否、未执行 |
| 自评口径是否正确 | 是、否、未知 |
| 自评结果是否正确 | 是、否、未知 |
| 轮次 | 对话轮次或工具轮次 |
| 耗时 | 单题或整组耗时 |
| 失败类型 | 标准失败枚举 |
| 备注 | 假设、风险、错误、修复过程 |

## 产品链路证据

如果评测完整产品链路，额外记录：

- 数据连接证据。
- schema 或上下文构建证据。
- 检索、记忆或样例复用证据。
- 执行前校验证据。
- 查询或分析执行证据。
- 图表或报告生成证据。
- 权限治理证据。
- 错误修复证据。

## 最终报告结构

最终报告建议包含：

- 评测摘要。
- 实验组说明。
- 总览评分表。
- 能力模块评分表。
- 逐题评分表。
- 产品链路评分表。
- 关键题详解。
- 失败类型分析。
- 实验有效性限制。
- 结论。
- 下一轮建议。

## 报告标准

- 不只报总分。
- 必须解释胜负原因。
- 必须说明实验限制。
- 必须区分模型能力、上下文能力和产品链路能力。
- 必须列出关键失败案例。
- 必须说明哪些能力可落地，哪些仍需人工复核。
